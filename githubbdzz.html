针对Github屏蔽百度爬虫导致博客无法收录解决办法

发表于 2018-05-11 更新于 2020-10-01 分类于 Github ， SEO

Github屏蔽百度蜘蛛(BaiduSpider)

我的博客搭建到现在也有一段时间了，今天忽然发现百度对于我博客的收录没有几条，使用百度站长抓取诊断工具测试了几个链接，发现都是抓取失败返回信息403。

联系了Github的用户服务部门，对方的答复是百度爬虫爬得太猛烈，所以就禁掉了……

问了一下曾经从事这方面的朋友，经过他的提醒，想起在几年前发生的的一个事：百度统计js被劫持用来DDOS Github。关于这个事网上有很多记录，我会把它贴到文章末尾。

好吧！既然Github已经彻底拉黑了百度蜘蛛，Github客服也无法提供帮助，为了让更多人能看到我的博客，只好想想其他解决办法。

博客无法被收录解决

想要解决这个问题，最好的办法就是不要让百度爬虫直接访问Github，需要在中间套一个反向代理。那么新的问题就来了，既然我们可以搭建个反向代理服务器，为什么不直接把博客搭建到服务器上？本来我们把博客托管在Github不就是为了尽可能的减少成本吗？

So 这个方案Pass。

我们还可以去找一些第三方的反向代理服务器。比如现在常见的CDN服务，不仅可以当作反向代理使用，还附带了各种网络环境下的加速服务。CDN和反向代理有一个很大的区别就是它会做缓存，并向各个节点服务器分发。还有一个就是CDN只给静态资源做，不过这个对我们来说不是问题，因为本来我们的博客不就是纯静态的么。下面写几种常见的CDN服务。

利用CDN服务进行解决

现在常见的CDN服务商有以下几种：

这几个CDN服务商的对比下，又拍云是比较符合我的需求的。下面介绍一下各个CDN的功能。

加速乐

加速乐其实是一个DNS解析服务商。你需要把自己域名的NS地址改成它的，才能使用加速功能。除了访问加速、页面压缩、智能刷新，还有防盗链、网站防火墙、DDOS防火墙、CC防火墙等防黑功能。

但加速乐的问题在于它并不是单纯的CDN服务商，她的加速和CDN还是有很大的区别：加速乐首先会进行判断内容是否应该被加速(比如后缀是不是JS、CSS、HTML等静态资源)，如果博客设置的文章地址是没有后缀的例如：**2018/05/wenzhang/**就不一定会被加速。并且加速乐也不是百分百加速，可能会去根据各种情况进行加速或不加速。

七牛云存储

七牛就挺不错，它还提供了一个镜像存储功能。可以设置镜像源进行镜像存储。

使用七牛需要把网站的CNAME设置为七牛的地址，然后配置一下镜像源。

这样当我访问www.ly554.com/winadmin.html的时候，会把请求发送到七牛服务器，七牛会自动访问我设置的镜像源的内容，帮我缓存，并返回给我。

但是用过后又发现了一个问题，就是七牛只会帮我缓存我发送了访问请求的地址，不会自动进行缓存我的新文章，看了一下官方文档，七牛的镜像存储只是用于当作迁移工具使用，不会自动刷新缓存。

又拍云

其实我对又拍的印象并不是很好，以前我兼职卖鞋的时候用它存过照片，反正就是用的很不爽。

不过近几年又拍在云服务领域做的还不错，又拍云有和七牛同样的功能，但是人家的名字叫做CDN回源配置。而且有设置缓存时间的功能，上面明确写了：CDN缓存时间是指UPYUN回源取得文件后，文件在UPYUN CDN网络中的缓存的时间，超过缓存时间则重新回源获取文件。

这个功能就是我们所需要的了，可以自动更新缓存。

如果愿意花一点钱简简单单解决Github博客无法被百度收录的问题可以选择收费的第三方CDN服务，但要看好有没有缓存功能。

不过我还是想再省点钱，所以我最终使用了下面的办法。

同时在Coding发布博客用于国内访问

只需要同时把博客发布在Github和Coding上，然后设置域名解析国内线路解析到Coding，国外线路解析到Github。这样百度抓取的时候就会抓取到Coding上面的数据。

如何在Coding新建项目并部署，往下看。

在Coding新建项目并部署博客

创建Coding账号，用微信登陆或者邮箱注册自己开心就好。都是中文，这里就不详细说了。

新建Coding项目

注册好后在Coding个人中心点击创建新项目，项目名称和项目描述随意填写，项目类型选择公开，勾选使用README.md初始化项目。

Pages服务

注册好Coding项目后，点击Pages服务，开启Pages服务，部署来源设置为master分支，然后就可以看到你的Coding二级域名了。

绑定自定义域名需要完善一下个人信息，真麻烦。

配置ssh密钥

鼠标放在右上角头像处，点击个人设置，点击SSH公钥，新增公钥，公钥名称随便填(不填也行)，

之前我们在创建Github时已经创建过SSH密钥，打开你电脑的C:\Users\用户名.ssh文件夹，打开id_rsa.pub(公钥)文件，将里面所有内容复制，粘贴到公钥内容中。

永久有效勾不勾选看你心情，不勾选的话到指定日期公钥就失效了，需要重新设置。都设置好后点击添加，这样公钥就添加好了。
