使用Hexo搭建静态博客并托管在GitHub

发表于 2018-04-16 更新于 2020-10-01 分类于 Hexo ， GitHub

前言

      对于喜欢自行搭建博客的朋友们，对于一些BlogCMS系统一定不会很陌生，比如EM、PW等。我自己曾经建立过几次博客，第一次是使用Emlog，但是后来因为一些外部因素没有继续下去。第二次使用Wordpress搭建了博客，但是使用低价的空间进行搭建又不是很流畅。最后还是狠心抛弃了它。这次搭建的新博客不打算恢复以前的文章，一切从0开始吧.
      使用GitHub和Hexo进行搭建静态博客既节省了购买空间或服务器的费用，访问速度也可以支持日常使用。并且免去了安全维护的环节。
      GitHub为我们提供了一个不限大小的免费空间（个人建议使用不要超过1GB）。
网上对Hexo的评价是：Hexo的出现从某种意义上来说代替了Jekyll，让我们可以更专注于写作本身。
      本文主要介绍我在搭建博客时的过程和掉过的几个坑，网上的搭建教程多半都并不是很完善，所以博客的第一篇文章便写一下Hexo+GitHub搭建博客的过程。原理和其他衍生知识推荐大家去看官网的手册，其他推荐参考内容我会在文中写出来。
      写文章是通过Markdown语法进行写作，过段时间我会整理一篇Markdown常用语法出来。

阮一峰 - 喜欢写Blog的人，会经历三个阶段
第一阶段，刚接触Blog，觉得很新鲜，试着选择一个免费空间来写。
第二阶段，发现免费空间限制太多，就自己购买域名和空间，搭建独立博客。
第三阶段，觉得独立博客的管理太麻烦，最好在保留控制权的前提下，让别人来管，自己只负责写文章。

      我可能是第四阶段，嫌弃免费空间限制多速度慢又感觉收费空间不划算，并且觉得独立博客管理太麻烦。Ps:主要是，没钱 (°ー°〃)

Hexo简介

Hexo is a fast, simple & powerful blog framework

      Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

绑定域名

Github绑定域名有三种方法：

一：A记录：域名直接映射IP，直接将域名设置A记录指向Github提供的域名主机IP。

二：如果域名提供商支持ALIAS或ANAME，将域名指向xxxx.github.io，这样可以在域名解析得到一个访问最快的动态IP。

三：CNMAE：将域名配置成CNAME，指向xxxx.github.io，这样可以在域名解析的时候得到一个动态的IP，这个IP是一台离你最近的镜像主机。

其中第二、三种方法可以享受到Github提供的CDN加速(xxxx代表你的Github用户名)

我用的是第三种方法，

  • 先去将要绑定的域名设置CNAME记录值指向到你的GitHub地址，例如：xxxx.github.io
  • 然后去GitHub，点击进去你创建的那个博客仓库点击create a new file
  • Name your file为CNAME
  • 内容填写你自己申请的域名，如xxxx.com
  • 然后提交
  • 点击Clone or downliad，点击Download ZIP，把里面的CNAME文件拿出来放到Hexo文件夹中的Source目录下

搜索引擎认为带www和不带www域名是两个不同的网站，如果带www和不带www的域名指向同一个站点，内容相同，是会导致权重分散的。
Github支持301重定向，如果你把CNAME中填写的是一级域名，并把二级域名也指向到xxxx.github.io，访问二级域名会301重定向到CNAME中填写的一级域名。从而不会导致权重分散。
